# Visibility Protocol Removal: Summary

## What changed

Visibility (private / unlisted / public) was previously modelled as a first-class field in the handprint protocol and CLI. It has been removed entirely. The hub (handprint.sh) is now the sole owner of visibility; it defaults new handprints to unlisted.

## Files changed

### `packages/types/src/handprint.ts`
- Removed `VISIBILITY_LEVELS` constant array.
- Removed `visibilitySchema` Zod enum.
- Removed `Visibility` TypeScript type.
- Removed the "Visibility constants" comment section.

### `packages/types/src/profile.ts`
- Removed `VISIBILITY_LEVELS` and `visibilitySchema` from the import of `./handprint.js`.
- Removed the `visibility` field from `socialLinkSchema`.
- Removed the `visibility` field (with its default) from `projectConfigSchema`.

### `packages/types/src/__tests__/profile.test.ts`
- Replaced "defaults visibility to private" test with "accepts config with no visibility field".
- Removed the `visibility: 'private'` fixture value from the first test.

### `src/dirs/project.ts`
- Removed `Visibility` from the type import.
- `initProject` no longer accepts a `visibility` parameter.
- The written `config.json` no longer contains a `visibility` field.

### `src/commands/init.ts`
- Removed `Visibility` type import.
- Removed `visibility` from the `options` parameter of `init()`.
- Calls to `initProject` no longer pass a visibility argument.

### `src/commands/push.ts`
- Removed `loadProjectConfig` import (no longer needed).
- Removed the `visibility: 'private'` early-return gate.
- `PushResult` no longer has a `visibility` field.
- Return values no longer include a visibility string.

### `src/commands/status.ts`
- Removed `loadProjectConfig` import.
- `StatusResult` no longer has a `visibility` field.
- The result object no longer initialises or populates `visibility`.

### `bin/handprint.ts`
- Removed `Visibility` type import.
- Removed the `--visibility <level>` option from the `init` command.
- Removed the `if (result.visibility === 'private')` branch from the `push` action.
- Removed the `visibility:` line from the `status` output.

### `tests/commands/status.test.ts`
- Removed `expect(result.visibility).toBe('private')` assertion.

### `tests/dirs/project.test.ts`
- Replaced two visibility tests ("defaults to private", "respects visibility param") with a single test confirming no `visibility` key exists in the parsed config.

### `README.md`
- Added a "Visibility" section explaining visibility is a hub concept, not part of the protocol or signed record, with the three levels and where to set them.

### `AGENTS.md`
- Added a "Visibility" section (section 6) with the same explanation, renumbering the previous section 6 to 7.

### `CHANGELOG.md`
- Added an `[Unreleased]` entry documenting all changes.
