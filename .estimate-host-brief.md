## 1. Rough token estimate (GrabPlan.estTokensIn)
In `src/commands/grab.ts`:
- Add `estTokensIn: number` to the `GrabPlan` interface.
- During the scan loop, accumulate content chars for the sessions that will be processed: for each session's `fresh` entries, add `Math.min(e.text.length, 1000)` (1000 = the ENTRY_TEXT_MAX cap used by the window builder; add a brief comment). Call the accumulator e.g. `contentChars`.
- Import `SYSTEM_PROMPT` from `../extractor/prompt.js`. Compute `estTokensIn = Math.round((contentChars + totalChunks * SYSTEM_PROMPT.length) / 4)` (a ~4 chars/token heuristic; it is a rough input-token estimate, document as rough). Put it on the plan.

## 5. CLI display (bin/handprint.ts) — the decision info

- A branded extractor line: if the label starts with `host:`, show `Extractor: <label> (<Brand>)`; if it starts with `ollama:` or `local:`, show `Extractor: <label> (on your machine)`.
- An estimate line:
