## How Auto-resync Decides to Copy

`ensureSkillSynced` (called at CLI startup, errors swallowed):

1. If `HANDPRINT_NO_SKILL_SYNC` env var is set: return immediately (no-op).
2. Resolve `claudeHome` (default `~/.claude`). If the directory does not exist: return (never create `~/.claude` implicitly).
3. Read `<claudeHome>/skills/handprint/.version`. If SKILL.md is missing OR the recorded version != `cliVersion()`: call `installSkill({scope:'global'})`. Otherwise: no-op (fast path, no file I/O beyond a stat + one small read).

This means:
- First install after `npm i -g` with an existing `~/.claude`: copies the skill.
- Subsequent runs at the same version: stat + read .version (effectively free), no copy.
- After `npm i -g handprint-sh@latest`: version mismatch triggers one copy, then no-ops forever.

## End-to-end Verification (dist/bin/handprint.js)

With a temp HOME directory and `~/.claude` pre-created:
- `handprint skill install`: writes `~/.claude/skills/handprint/SKILL.md` and `.version` containing `0.4.5`. Prints path.
- `handprint skill uninstall`: removes `~/.claude/skills/handprint/`. Prints confirmation.
- `handprint --version` (auto-resync): with `~/.claude` present and no skill installed, the auto-resync on startup installs the skill before printing the version.
